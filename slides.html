Welcome!

What is version control?

Version control refers to any system that manages changes to files over time.

This makes refering to – and reverting back to – earlier stages of a project very easy.
It also makes collaborating with others easier since you can understand what changes were made and why.

Why is this tutorial in two parts?

For most use cases, interacting with git locally, via the command line, is preferred, but there are two use cases (perhaps more) where it would reasonable to do everything using GitHub’s web interface.

(1) You have an immediate need to share your code online as a static “final product.”

(2) You want to create a GitHub Pages website that is relatively simple (e.g., by cloning someone else’s theme and making small infrequent edits to it).
